 我来帮你设计一个"句子迷"风格的Web前端项目。这种静态托管+GitHub数据存储的方案非常适合个人内容收藏。

## 核心设计思路

### 1. 技术架构
```
┌─────────────────┐     ┌──────────────────┐
│   GitHub Pages  │────▶│  GitHub API      │
│  (前端托管)      │     │  (获取MD/JSON数据) │
└─────────────────┘     └──────────────────┘
         │
         ▼
┌─────────────────┐
│  utterances /   │
│  giscus 评论系统 │  ← 基于GitHub Issues的评论
└─────────────────┘
```

### 2. 文件结构设计
```
sentences-collection/
├── index.html          # 主页面
├── data/
│   ├── sentences.json  # 句子数据
│   └── categories.json # 分类配置
├── assets/
│   ├── css/style.css
│   └── js/app.js
├── .github/
│   └── workflows/      # 可选：GitHub Actions更新数据
└── CNAME               # 自定义域名
```

### 4. 界面设计建议

**整体风格**：极简、留白、书卷气，类似[句子迷](https://www.juzimi.com/)但更现代

**核心页面布局**：

```
┌─────────────────────────────────────┐
│  [Logo]  每日一句  分类  关于  [搜索] │
├─────────────────────────────────────┤
│                                     │
│     ┌─────────────────────────┐     │
│     │                         │     │
│     │   "句子的内容..."        │     │
│     │                         │     │
│     │         —— 作者          │     │
│     │                         │     │
│     │   [复制] [喜欢] [分享]    │     │
│     └─────────────────────────┘     │
│                                     │
│   [瀑布流/卡片式展示更多句子]          │
│                                     │
├─────────────────────────────────────┤
│  GitHub Comments (utterances)       │
└─────────────────────────────────────┘
```

**关键交互细节**：
- **卡片悬停效果**：轻微上浮+阴影，显示操作按钮
- **分类筛选**：标签云或侧边栏，支持多选
- **随机浏览**：首页"每日一句"可刷新随机展示
- **复制功能**：点击句子直接复制到剪贴板
- **暗黑模式**：阅读体验更舒适

### 5. 评论系统集成

推荐使用 **giscus**（基于GitHub Discussions）或 **utterances**（基于GitHub Issues）：

**配置步骤**：
1. 安装 giscus app 到你的仓库
2. 开启 Discussions 功能
3. 在前端嵌入脚本：
```html
<script src="https://giscus.app/client.js"
        data-repo="yourname/sentences"
        data-repo-id="..."
        data-category="句子评论"
        data-mapping="pathname"
        ...>
</script>
```

### 6. 数据维护方案

**方案A - 直接编辑**：
手动编辑 `data/sentences.json` 提交到GitHub

**方案B - GitHub Issues 驱动**：（这个好，谁都可以参与）
用 Issues 模板提交新句子，Action 自动更新 JSON 文件

**方案C - 本地脚本**：
写一个 Python/Node 脚本管理本地数据，一键推送

## 进阶功能建议

1. **PWA支持**：可离线浏览收藏的句子
2. **图片分享**：一键生成精美的句子图片（canvas/svg）
3. **阅读统计**：用 localStorage 记录阅读历史
4. **响应式**：移动端优化，支持滑动切换
5. **接入真实 GitHub API** 获取数据
6. **生成分享图片**（canvas 把句子转成精美图片）
7. **PWA 离线缓存**
8. **自动生成 GitHub Issues** 来收录新句子
